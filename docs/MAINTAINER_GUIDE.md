# Maintainer Guide

## Rollout Tasklist

- [ ] Publish the repository publicly.
- [ ] Configure repo metadata from `docs/GITHUB_SETUP.md`.
- [ ] Publish wiki pages from `wiki/`.
- [ ] Seed labels from `docs/github-labels.yml`.
- [ ] Create selected issues from `docs/seed-issues/`.
- [ ] Invite candidates as temporary collaborators.
- [ ] Have candidates claim issues in comments.
- [ ] Require PR template completion.
- [ ] Review code plus explanation quality.
- [ ] Remove collaborator access after the exercise.

## What to keep private

Do not publish:
- hidden evaluation scenarios,
- private reviewer-only tests,
- internal scoring notes,
- or interviewer prompts you want to preserve as control checks.

A private bundle with those materials is included separately in the delivery zip.
