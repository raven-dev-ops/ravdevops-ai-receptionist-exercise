# Candidate Handbook

## Core rule

AI is allowed. Opaque submissions are not.

## What to do

1. Claim an issue.
2. Create a branch.
3. Implement the scoped change.
4. Run tests and repo checks.
5. Open a PR using the required template.
6. Be ready to explain the code.

## Review emphasis

The best submissions are:
- correct,
- narrow in scope,
- well-tested,
- and easy to explain.

## Anti-patterns

- full rewrites for small issues
- code you cannot defend
- undocumented behavior changes
- hidden dependency on hosted services
